汇编中if与switch的效率区别 ​

最近在学习x64汇编的时候，看到C语言的if和switch底层实现有很大的不同，于是就猜测Java是否也是采用相同的策略，经过反汇编后，验证果然如此，但我这里还是先讲C语言版本的，因为Java反汇编后是Java自己面向JVM的字节码格式不好演示，不过基本的if和switch都一样。

if else语句： ​

c
void main(){
    int a = 0;
    if (a == 1){
        a = 1;
    }else if(a == 2){
        a = 2;
    }else {
        a = 3;
    }
}

可以看到，如果条件不成立，则会继续往下判断，直到结束为止，也就是说，如果我们有N个if，极端情况下，就需要判断N次，从时间复杂度来算是O(n)。

switch语句： ​

c
void main(){
    int a = 0;
    switch(a){
        case 1:
            a = 1;
            break;
        case 3:
            a = 3;
        case 4:
            a = 4;
            break;
        case 5:
            a = 5;
            break;
        case 7:
            a = 7;
            break;
        default:
            a = 5;
    }
}

需要注意的是：我特意在case3中没有写break语句以及其匹配值并不是连续的，然后看看接下来生成的汇编会是什么样。

将其翻译为伪代码来描述其逻辑。

c
void main(){
    int a = 0;
    if(a > 7){
        goto defalut;
        return;
    }
    // 生成一个地址的指针数组，作为跳转表。
    *address[8] = {'default', 'case1', 'default', 'case3', 'case4', 'case5', 'default', 'case7'};
    // 无条件跳转到address[a]的地址
    goto address[a]; //比如这里 a = 0,于是跳转到default;
}

可以看到，编译器帮我们优化了代码，将其逻辑改成了生成一个跳转表，然后根据其下标直接进行跳转，而无需IF语句一个一个判断，

这也就是说，无论多长的case，我们都只需要最多跳转1次，就可以搞定。

另外有趣的现象在于，case匹配只有5个，但最终却生成了的跳转表有8个，而我们正好缺case 0、case 2、 case 6，这些下标被自动填充了default，表示没有匹配到的。

此外，在上图可以看到当在case3里没写break的时候就不会生成jmp语句，这也就是为什么不写break会接着往下执行的底层逻辑。

java语言对于IF和switch也是相同的处理。 ​

IF语句 ​

java
package org.zhaoshun.domain;
public class Test {
	public static void main(String[] args) {
		int a = 0;
		if (a == 1){
			a = 1;
		}else if(a == 2){
			a = 2;
		}else if(a == 3){
			a = 3;
		}else if (a == 4) {
			a = 4;
		}else {
			a = 5;
		}
	}
}

不过Java的汇编是自己面向JVM的格式，说实话我看不懂，不过靠着源代码和英文还是能猜出来。

从上面反汇编逻辑可以看出，其逻辑也是有多少次判断多少次，直到为true或结束为止

再来看看switch是如何实现的： ​

java
package org.zhaoshun.domian;
public class Test2{
	public static void main(String[] args) {
		int a = 0;
		switch(a){
			case 1:
				a = 1;
				break;
			case 2:
				a = 2;
				break;
			case 3:
				a = 3;
			case 5:
				a = 4;
				break; 
			default:
				a = 0;
		}
	}
}
