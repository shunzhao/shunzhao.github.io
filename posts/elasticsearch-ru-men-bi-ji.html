elasticsearch入门笔记 ​

Elasticsearch 是一个分布式、RESTful 风格的搜索和数据分析引擎，也可以理解为一个分布式的数据库。

最常见的使用方案就是在搜索时需要全模糊查询时使用，某 JAVA 圣经规范中也明确说明了：

【强制】页面搜索严禁左模糊或者全模糊，如果需要请走搜索引擎来解决。 ​

说明： 索引文件具有 B-Tree 的最左前缀匹配特性，如果左边的值未确定，那么无法使用此索引，全模糊也会失效。全模糊使用 elasticsearch 才能解决这个问题

相关名词解释 ​

Node 和 Cluster ​

Elasticsearch 是一个分布式的系统，一个 Node 是一个节点， Cluster 则是由多个节点组成的集群。

Index ​

可以理解为数据库中的 Database 。

document ​

可以理解为数据库中的 row 一行记录。

doc （Type） ​

可以理解为数据库中 Database 下的 Table，原先叫 type，其作用是为了分组对 document 进行 分组 在 elasticsearch 8.0版本之后被废除，改为叫 doc，并且一个 Index 只有 一个doc

安装 ​

官网下载安装。

正常启动后，默认运行在 9200 端口。

使用 ​

HTTP REST API 模式 ​

创建 Index ​

请求：

http
PUT localhost:9200/<indexName> 
// <indexName> 替换为实际的索引名称

响应：

json
// 成功创建后会返回下列信息
{
    "acknowledged": true,
    "shards_acknowledged": true,
    "index": "wocao"
}

删除 Index ​

请求：

http
DELETE localhost:9200/<indexName>
// <indexName> 替换为实际的索引名称

响应：

json
// 成功创建后会返回下列信息
{
    "acknowledged": true
}

往创建的 Index 中添加数据 ​

请求：

http
PUT、POST localhost:9200/<indexName>/_doc/<id>
// <indexName> 替换为实际的索引名称
// id 可以自己指定，也可以不指定自动生成
// 请求体为 JSON 格式
// 例子：

{
    "name": "张三",
    "skill": "java开发、数据库管理"
}

响应：

json
{
    "_index": "employ",
    "_id": "zjAQxoMB9BUQJylZ77Wm",
    "_version": 1,
    "result": "created",
    "_shards": {
        "total": 2,
        "successful": 1,
        "failed": 0
    },
    "_seq_no": 27,
    "_primary_term": 2
}

更新数据： ​

注意看创建数据的响应体，其中需要重点关注的字段有：result 为 created 并且 _version 为 1。

用上面的数据原封不动的再次发起请求

响应：

json
{
    "_index": "employ",
    "_id": "123332131",
    "_version": 2,
    "result": "updated",
    "_shards": {
        "total": 2,
        "successful": 1,
        "failed": 0
    },
    "_seq_no": 40,
    "_primary_term": 2

再次看 result 为 updated 并且 _version 为 2。

创建连接 ​

java
@Bean
ElasticsearchClient initElasticsearchClient() {
    // 创建低级客户端
    RestClient restClient = RestClient.builder(
        new HttpHost("localhost", 9200)).build();
    // 使用Jackson映射器创建传输层
    ElasticsearchTransport transport = new RestClientTransport(
        restClient, new JacksonJsonpMapper());
    // 创建API客户端
    return new ElasticsearchClient(transport);
}

模糊搜索查询 ​

匹配单个关键字查询 ​

java
@Test
void searchByMarch() throws IOException {
    // 在名为 employ 的 index 中搜索字段名为 skill 中包含"数据库"的数据
    SearchResponse<Employ> response = this.elasticsearchClient.search(s -> s.index("employ").query(q -> q.match(t -> t.field("skill").query("数据库"))), Employ.class);
	// 查询到的数量
    TotalHits total = response.hits().total();
    System.out.println(total);
    boolean isExactResult = total.relation() == TotalHitsRelation.Eq;
    if (isExactResult) {
        log.info("There are " + total.value() + " results");
    } else {
        log.info("There are more than " + total.value() + " results");
    }
    // 获取查询到的数据
    List<Hit<Employ>> hits = response.hits().hits();
    for (Hit<Employ> hit : hits) {
        Employ employ = hit.source();
        log.info("Found employ " + employ.toString());
    }
}

多个关键字为 and 关系 ​
java
@Test
void searchByAnd() throws IOException {
    // 构建搜索条件
    Query type1 = MatchQuery.of(m -> m
                                .field("skill")
                                .query("java")
                               )._toQuery();

    Query type2 = MatchQuery.of(m -> m
                                .field("skill")
                                .query("数据库")
                               )._toQuery();
    // 在名为 employ 的 index 中搜索字段名为 skill 中包含"数据库" 和 "java"的数据
    SearchResponse<Employ> response = this.elasticsearchClient.search(s -> s.index("employ").query(q -> q.bool(b -> b.must(type1).must(type2))), Employ.class);
    // 获取查询到的数据
    List<Hit<Employ>> hits = response.hits().hits();
    // 遍历
    for (Hit<Employ> hit : hits) {
        Employ product = hit.source();
        log.info("Found employ " + product);
    }
}
